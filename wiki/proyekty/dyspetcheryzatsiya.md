# Диспетчеризація вантажів (міні-CRM)

Таблиця-CRM для операційного ведення вантажів: статуси, ETA, трекінг ліній.

## Де живе продакшн
- **n8n Cloud**: irynakarmazina.app.n8n.cloud (Google-акаунт unitex.automation@gmail.com).
- Майстер-таблиця Google Sheets (аркуші: Відправлення / Дашборд / Клієнти / Невідомі BL / Демередж).
- Код у репо: гілки `crm-dispatch` (`dispatch/`) і `claude/repository-visibility-08xxop`.

## Три дошки
- **Операційна** — робоче місце логіста.
- **Керівнича** — з фільтрами Менеджер · Операційний менеджер · Клієнт · Лінія.
- **Клієнтська вітрина** — клієнт бачить тільки своє (за токеном), точки маршруту за напрямком, без цін.

## Модель статусів (спільний словник, ЗАКРИТО)
Букінг · Стафіровка · В порту відправлення · Завантажений на судно · В морі · Вивантажений
в порту прибуття · (імпорт далі) Завантажений на авто/потяг · Вантаж доставлено.
- ІМПОРТ = 8 статусів (заморозка на «Вантаж доставлено»).
- ЕКСПОРТ = перші 6 (заморозка на «Вивантажений в порту прибуття»).
- Технічні кроки (драфт BL, телекс, документи, ДО, Т1, митниця) — окремі робочі колонки, НЕ в «Статус».

## Ключ трекінгу (погоджено)
Є BL → по BL (Maersk, 9 цифр); немає BL, є контейнер+лінія → тимчасово по контейнеру з позначкою;
з'явився BL → автоперехід; немає обох → список «не трекається».

## Автоматизація
- **Тільки Maersk** — через API (розклад 07:00 Києва). MSC і CMA CGM — ВРУЧНУ людиною
  (CMA має анти-бот DataDome; рішення: без автоматизації цих ліній).
- Правила: оплачено ≠ доставлено; потяг = carrier haulage; ручні дані не перетирати мовчки → коментар.

## ⚠ Ключові уроки / ризики
- **Не інферити «доставлено» зі статусу «Завантажений на авто/потяг»** — автомат ставить його
  і на останню милю, і на довіз У ПОРТ ВІДПРАВЛЕННЯ; «Вантаж доставлено» ставить ЛЮДИНА.
  Асиметрія: хибне «в роботі» безпечне, хибне «доставлено» небезпечне (ховається зі списку).
- **Безпека**: керівничий дашборд-вебхук n8n БЕЗ авторизації (270 КБ даних будь-кому з посиланням) —
  вразливість підтверджена наживо. Перевипустити ключ Maersk API (був відкритим текстом).

## Відкриті питання
- ~30 Maersk-BL без статусу (429 rate-limit); розбивка багатоконтейнерних BL; тарифи демереджу (чернетка).
- Джерело нових угод — автозавантаження з 1С при передачі угоди в роботу (синергія з [[1c-api]]).

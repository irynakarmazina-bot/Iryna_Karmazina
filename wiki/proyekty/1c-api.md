# Доступ до 1С Експедитор (API)

Джерело фінансових і операційних даних. Потрібне для [[finzvit]] і [[dyspetcheryzatsiya]].

## Поточний стан
- Пряма публікація бази (OData) НЕДОСТУПНА: хостер RDPort ВІДМОВИВ.
- Тимчасово — ручне вивантаження журналів з 1С (експорти доходних/витратних/каси/угоди).
- Є розробник Експедитора — йому передано ТЗ на API; чекаємо адресу і доступи.
- План Б у резерві: RDP-агент з VPS + прокид диска (xfreerdp /drive) → вивантаження у файл на VPS.

## ТЗ для розробника (що просимо через API)
Тільки на читання, JSON по HTTPS, ~1 раз на день:
1. **Доходні рахунки** (всі за період, з полями: номер, дата, клієнт, угода, сума, валюта, сума У.О.,
   статус оплати, дата оплати факт, ознака авансу, рядки зі статтями, контейнери).
2. **Витратні рахунки + аванси постачальникам** — аналогічно + постачальник, **поле «Борг»** (обов'язкове).
3. **Залишки банк/каса** — вид оплати, валюта, залишок.
4. **Угоди** — номер, дата, клієнт, статус, суми доходних/витратних.

## ⚠ Критичні факти про дані Експедитора
- «Запланована дата оплати» — **НЕПРАВИЛЬНА** (автозаповнення). Прогноз/aging будувати
  ТІЛЬКИ від фактичних термінів (див. [[dso-dpo]]).
- Каси НЕ взаємозамінні (див. [[likvidnist-po-kasah]]).
- Розрізняти виставлено/оплачено на КОЖНОМУ кроці; поле «Борг» обов'язкове.
- Звіт ведеться в У.О. (USD-еквівалент); 13 кас у 3 валютах (див. [[kasy]]).
- Повний мапінг колонок експортів — [[mapping-1c]].

## Доступ
RDP до сервера 1С — деталі доступу НЕ у вікі (тільки у користувача).
Read-only користувача в Експедиторі створює користувач сама.

# Оперативний фінансовий звіт

Звіт про поточний фінансовий стан Юнітекса з даних 1С Експедитор.
Живе ОКРЕМИМ файлом Google Sheets «Оперативний фінансовий звіт».

## Призначення
Показати стан на зараз: незакриті угоди, неоплачені рахунки, дебіторка/кредиторка,
ліквідність по касах, касовий розрив на 30 днів. Оновлення — щоранку автоматично
(мета) + за потреби. Бот для цього НЕ потрібен — скрипт + Google Sheets.

## Одиниця обліку
Звіт ведеться в **У.О. (USD-еквівалент 1С)**; оригінальні валюти — в деталізації
(валюти змішані: UAH / USD / EUR).

## Блоки дашборда (компоновка v8/v9)
- Залишки по касах · Ліквідність · Кешфлоу (факт назад)
- [[zamorozheno-v-ugodah|Заморожений капітал]] · Дебіторка · Оплати постачальникам
- Платіжний календар (прогноз вперед) · Діаграма
- Аркуш «КЛІЄНТИ» (див. [[klienty]]), «Знімки» (щоденний KPI → динаміка за тиждень)

## Реалізація
- Генератор-прототип: `report/build_report.py` на гілці `claude/financial-reporting-system-i6auod`.
  Захардкоджені TODAY і шлях — для VPS/API треба параметризувати дату/папку/джерело.
- Дашборд: Apps Script у Sheets — меню «📊 Звіт → Відкрити дашборд» (Chart.js, модальне вікно).
  `report/Code.gs` + `report/Dashboard.html`.
- ⚠ Попередній «звіт за період» у Sheets теж має Apps Script кнопку — при об'єднанні не перетерти.

## Відкриті питання
- Місячний прогноз надходжень через ETA контейнерів (лаг «ETA→рахунок» рахувати з файла b61b4230).
- Google Drive Юнітекса: готовий звіт вивантажувати самостійно (спершу search_files — оновити наявний
  файл, не плодити копії), якщо конектор працює. Перевіряти на старті сесії.
- Джерело даних: поки ручне вивантаження з 1С; мета — [[1c-api]].

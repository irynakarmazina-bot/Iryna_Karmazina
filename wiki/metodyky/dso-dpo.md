# Методика: реальні терміни оплати (DSO / DPO)

Скільки днів насправді минає від виставлення рахунку до оплати.
- **DSO** — по клієнтах (як швидко нам платять).
- **DPO** — по постачальниках (як швидко платимо ми).

## ⚠ Головне правило
«Запланована дата оплати» в 1С Експедитор — **НЕПРАВИЛЬНА** (автозаповнення).
На ній НЕ МОЖНА будувати прогноз чи aging.

## Як рахувати
```
очікувана дата оплати = дата рахунку + середній фактичний термін клієнта/постачальника
```
- Середній термін = різниця між датою виставлення і фактичною датою оплати
  (по історії оплачених рахунків цього контрагента).
- Без історії по контрагенту — медіана по всіх.
- Динаміку рахуємо самі щоденними знімками на VPS (історію від API не вимагаємо).

## Статус як «стабільний факт»
DSO/DPO — похідні від живих даних, змінюються повільно. Авторитетне значення завжди
рахує [[finzvit]] зі знімків; у вікі (напр. [[klienty]]) можна тримати поточну оцінку
з поміткою «станом на <дата>», але істина — у перерахунку.

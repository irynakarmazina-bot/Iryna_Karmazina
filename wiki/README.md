# Wiki — база знань (LLM Wiki)

Це персональна LLM-вікі за патерном Андрія Карпаті (див. відео Ігоря Матрофайла
«Як я побудував власного AI, який пам'ятає все»). Мета: єдине місце, де зберігаються
стабільні знання про бізнес Юнітекса й мої проєкти, звідки і бот, і Claude у сесіях
можуть швидко діставати інформацію природною мовою.

## Три шари
- **Інтерфейс** — ці markdown-файли (читаються будь-де: VS Code, GitHub, Obsidian).
- **Двигун** — Claude (у сесіях і на VPS): читає, класифікує, оновлює сторінки.
- **База** — сама вікі: `raw/` (сире) + теми (розкладене по сутностях і методиках).

## Структура папок
```
wiki/
├── README.md        ← цей файл: правила ведення вікі (схема)
├── overview.md      ← високорівневий огляд: мапа гілок, ключові факти, відкриті питання
├── index.md         ← каталог усіх сторінок (1 рядок на кожну)
├── log.md           ← хронологія змін вікі (append-only)
├── raw/             ← сирі джерела (не редагувати руками)
├── proyekty/        ← проєкти: бот, фінзвіт, диспетчеризація, 1С-API
├── metodyky/        ← методики й правила розрахунків
├── klienty/         ← клієнти
├── postachalnyky/   ← постачальники
└── kasy/            ← каси/рахунки (види оплати)
```

## Правила ведення (схема) — для Claude
1. **Тільки факти.** Нічого не вигадувати. Джерело кожного факту — MEMORY.md / реальні
   дані / рішення користувача. Немає даних — писати «даних немає».
2. **Жива фінансова цифра НЕ у вікі.** Тут — методики й стабільні факти («як рахувати»,
   «Громикс платить у сер. за N днів станом на дату»). Актуальні суми (дебіторка, гроші
   по касах тощо) рахує Оперативний фінансовий звіт з даних 1С — у вікі їх не дублювати.
3. **Паролі/токени/доступи — ніколи.** RDP, Maersk API, n8n, ключі — не у вікі
   (і не в репо; тільки .env на VPS / n8n Credentials).
4. **Нічого не видаляти — переносити.** Застарілий факт не стирати, а позначати
   «станом на <дата>» або переносити в raw/, лишаючи слід.
5. **Зв'язки за змістом.** Сторінки пов'язувати `[[wiki-посиланнями]]` за суттю
   (тема стосується теми), не за назвою файлу.
6. **При зміні вікі** — оновити `index.md` (якщо додалась/змінилась сторінка) і
   дописати рядок у `log.md` (формат: `## [YYYY-MM-DD] дія — опис`).

## Статус
- **Фаза A (Рівень 1)**: ручна вікі, засіяна з MEMORY.md. Веде Claude у сесіях.
- Автоматизація (бот сам поповнює) і health-check — наступні фази, поки НЕ ввімкнені.

Первинне джерело для засіву: `../MEMORY.md` та `../MEMORY_ARCHIVE.md`.

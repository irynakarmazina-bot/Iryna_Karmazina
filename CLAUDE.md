# Контекст для Claude — прочитай перед будь-якою дією

## ❗ Головні правила (найвищий пріоритет, діють у КОЖНІЙ сесії)
1. НІЧОГО не робити, чого користувач не просив або не підтвердив. Питання «чи можеш зробити X?» = відповідь + уточнення + варіанти, БЕЗ коду і змін. Код і зміни — лише після явного «так».
2. Тільки факти: без припущень, вигадок, емоцій і замовчувань. Кожна цифра має показуватись розбивкою. Немає даних — сказати «даних немає», а не вигадувати.
3. Паролі, токени й доступи НІКОЛИ не потрапляють у репозиторій, логи, таблиці, MEMORY.md чи повідомлення — тільки захищені сховища (.env на VPS, credentials n8n).
4. Готовий фінансовий звіт вивантажувати на Google Drive Юнітекса самостійно, не чекаючи прохання (спершу search_files — оновити наявний файл, не плодити копії), якщо Drive-конектор у сесії працює.
5. Пряма команда користувача важливіша за моє рішення. Якщо вона просить конкретну дію (напр. «подивись відео») — виконувати саме її, а не підміняти власним обхідним варіантом. Якщо виконати неможливо (блок, немає доступу, немає даних) — ЗУПИНИТИСЬ, чесно сказати «не можу, бо…» і дати варіанти, а НЕ робити щось інше «замість». Повторення команди = сигнал, що я її зігнорував: зупинитись і виконати.

## Чек-лист старту сесії (перші дії, до роботи над задачею)
1. Прочитати MEMORY.md: спочатку «📌 ПОСТІЙНИЙ БЛОК» угорі (мапа гілок, факти, відкриті питання), потім останні секції журналу.
2. Перевірити конектори, потрібні для задачі (Google Drive тощо): якщо потрібного конектора в сесії немає — ОДРАЗУ повідомити користувача одним реченням. Сесія сама підключити конектор не може — він вмикається в налаштуваннях claude.ai (Settings → Connectors) і має бути доступний при створенні сесії.
3. Перевірити дату `last_memory_cleanup` (див. «Обслуговування MEMORY.md»).

## Як оновлювати MEMORY.md

Не чекати команди «онови пам'ять» і не чекати кінця сесії. Записувати одразу при:
- прийнятому технічному рішенні
- значимому коміті/пуші
- виявленому факті/обмеженні, яке знадобиться далі
- незавершеній задачі, яку треба передати наступній сесії

Кожне оновлення MEMORY.md робити тільки в гілці main:

```bash
current=$(git branch --show-current)
git checkout main && git pull origin main
# Edit MEMORY.md (нова секція в кінець)
git add MEMORY.md && git commit -m "memory: <опис>" && git push origin main
git checkout "$current"
```

## Мої проєкти

### Telegram-бот на Claude
Головний файл — bot.py. Залежності — requirements.txt.

**Сервер**
Бот працює на VPS DigitalOcean.
- Шлях: /root/Iryna_Karmazina
- Сервіс: mybot.service (systemd)
- Автодеплой: /root/autodeploy.sh запускається щохвилини, робить git pull і перезапускає сервіс при змінах.

**Git Relay**
Механізм для виконання команд на сервері через GitHub без термінала.
- `cmds/pending.json` — команда для виконання: `{"id":"...", "cmd":"..."}`
- `cmds/result.json` — результат виконання: `{"id":"...", "stdout":"...", "stderr":"...", "returncode":0, "ts":"..."}`
- `cmd_runner.py` — скрипт на сервері, читає pending.json кожні 5 секунд і виконує команди
- Сервіс: cmdrunner.service (systemd)

**Workflow**
Усе через GitHub: коміт у main → за хвилину сервер оновиться.
Руками до сервера не лізьмо.

## Обслуговування MEMORY.md

На початку кожної сесії перевіряй дату `last_memory_cleanup` в MEMORY.md.
Якщо минуло більше 30 днів — нагадай користувачу (одним реченням на початку відповіді):

> "До речі, минув місяць з останнього очищення MEMORY.md — варто прибрати старі записи."

**Як чистити MEMORY.md (раз на місяць) — НІЧОГО НЕ ВИДАЛЯТИ, тільки переносити:**
1. Відкрий MEMORY.md на гілці `main`
2. Секції журналу старші за місяць перенеси ДОСЛІВНО в MEMORY_ARCHIVE.md (у кінець, під заголовком «Перенесено YYYY-MM-DD»)
3. Перед перенесенням: незакриті задачі й важливі факти з цих секцій підніми в «📌 ПОСТІЙНИЙ БЛОК» (мапа гілок / факти / відкриті питання)
4. Онови `last_memory_cleanup`, закомить і запушай у `main`:
   ```bash
   git checkout main && git pull origin main
   # перенеси старі секції в MEMORY_ARCHIVE.md, онови постійний блок і last_memory_cleanup
   git add MEMORY.md MEMORY_ARCHIVE.md && git commit -m "memory: щомісячне перенесення в архів" && git push origin main
   ```

## Формат тіла листа (ти вписуєш ці рядки)

```
Авто: ФС1234 / ВТ5678
Дата: 14.07.2026
Перевізник: MULTYTRANS LLC
Перетин: Dorokhusk - Yahodyn
```
- **Авто** — до «/» тягач, після — напівпричіп → графа 25.
- **Дата** → графи 4 і 21. **Перевізник** → графа 16. **Перетин** → графа 13.

## Що заповнюється автоматично (з коносамента + інвойсу)

| Графа | Поле |
|---|---|
| 1 | Відправник (назва/адреса рядками) |
| 2 | Отримувач (назва/адреса рядками) |
| 3 | Місто + країна отримувача (DNIPRO / UKRAINE) |
| 4 | Порт розвантаження судна (GDANSK, POLAND) + дата з листа |
| 5 | Invoice No … Date: … |
| 6 | Номер контейнера |
| 7 | Кількість місць |
| 8 | Вид упаковки |
| 9 | Назва вантажу |
| 11 | Вага брутто |
| 12 | Об'єм |
| 13 | Пункт перетину (з листа) |
| 16 | Перевізник (з листа) |
| 21 | Місце + дата складання |
| 25 | Тягач / Напівпричіп (з листа) |

## Що робить

Лист (тема містить `CMR`/`ЦМР`, PDF-вкладення коносамент+інвойс) → Claude парсить **лише**
Bill of Lading і Commercial Invoice → заповнює шаблон → зберігає у папці CMR як
`CMR draft №[контейнер] [Компанія-отримувач]` → експортує PDF → відповідає на всіх з PDF.

## Що зробити тобі

1. Відправ тестовий лист (тема з `CMR`, вкладені коносамент+інвойс PDF, у тілі — 4 рядки формату вище).
2. У n8n глянь виконання: чи заповнилась чернетка правильно, чи прийшла відповідь.
   (IMAP-тригер, експорт PDF і SMTP окремо перевіряються тільки на реальному листі — якщо збій, напиши.)
3. Якщо ок — **Activate**.

## Технічна довідка

Розгортання/тести — з хмарної сесії через **Git Relay → VPS → n8n API**
(`N8N_API_KEY` у VPS `.env`, у чат не виводиться). Ядро (Claude-парсинг + заповнення) протестовано
наскрізь на реальних документах: усі поля вірні, залишкових плейсхолдерів немає.

# CMR draft — автоматизація n8n (Unitex)

Автоматичне створення чернетки CMR (міжнародної товарно-транспортної накладної)
з комерційних документів на вантаж.

## Що робить автоматизація

1. **Тригер** — новий лист на пошту Unitex, у **темі** якого є `CMR` або `ЦМР`.
2. Бере **PDF-вкладення** з листа.
3. **Парсить через Claude** — але **лише два документи**: `Bill of Lading` (коносамент)
   і `Commercial Invoice` (інвойс). Пакінг, сертифікати, декларацію — **ігнорує**.
4. Витягує поля:
   | Поле | Джерело |
   |---|---|
   | Номер контейнера | Bill of Lading |
   | Відправник (назва + адреса) | Bill of Lading / Invoice |
   | Отримувач (назва + адреса) | Bill of Lading |
   | Порт прибуття (Port of Discharge) | Bill of Lading |
   | Вага **брутто** (Gross Weight) | Bill of Lading |
   | Номер інвойсу | Commercial Invoice |
5. **Номери авто** бере з **тіла листа** у форматі `ФСХХХХ / ВТХХХХ`
   (до «/» — **тягач**, після — **напівпричіп**).
6. Копіює **шаблон Google Docs**, підставляє значення в плейсхолдери.
7. Зберігає новий файл на Google Drive у папці **CMR** з назвою
   `CMR draft №[номер контейнера] [Назва компанії отримувача]`.
8. Експортує його в **PDF** і **відповідає на лист** (на відправника + всіх у копії),
   прикріпивши PDF.

## Плейсхолдери в шаблоні

Файл `CMR_template.docx` — це твій бланк ЦМР із вставленими маркерами:

| Плейсхолдер | Графа ЦМР |
|---|---|
| `{{sender}}` | 1 — Отправитель |
| `{{receiver}}` | 2 — Получатель |
| `{{port}}` | 3 — Место разгрузки груза |
| `{{invoice}}` | 5 — Прилагаемые документы |
| `{{container}}` | 6 — Знаки и номера |
| `{{weight}}` | 11 — Вес брутто, кг |
| `{{truck}}` / `{{trailer}}` | 25 — Регистрац. номер (Тягач / Полуприцеп) |

## Разове налаштування (по кроках)

### 1. Шаблон → Google Docs
1. Завантаж `CMR_template.docx` у папку **CMR** на Google Drive Юнітекса.
2. Клік правою → **Відкрити за допомогою → Google Документи**.
   Google створить нативний Google Doc (плейсхолдери `{{...}}` збережуться).
3. Скопіюй **ID** цього Google Doc з адресного рядка:
   `https://docs.google.com/document/d/`**`ЦЕЙ_ID`**`/edit`
4. Встав цей ID у ноду **«Копія шаблону»** (поле File → By ID).

> ⚠️ Після конвертації відкрий Google Doc і переконайся, що таблиця ЦМР не
> «поламалась». Складні бланки Word іноді змінюють вигляд у Google Docs.
> Якщо вигляд не влаштовує — напиши, зробимо план Б (заповнення напряму в .docx).

### 2. Імпорт workflow
1. n8n → **Workflows → Import from File** → обери `CMR_draft_workflow.json`.

### 3. Credentials (доступи) — підключити в n8n
| Нода | Credential |
|---|---|
| Gmail — новий лист / Відповідь на лист | **Gmail OAuth2** (пошта Unitex) |
| Копія шаблону / Експорт у PDF | **Google Drive OAuth2** |
| Заповнення ЦМР | **Google Docs OAuth2** |
| Claude — парсинг | **Header Auth**: назва заголовка `x-api-key`, значення — твій ключ Anthropic |

> 🔒 Ключ Anthropic вписується **тільки** в n8n Credentials, ніколи — в цей файл чи репозиторій.

### 4. Дрібні перевірки
- Нода **«Підготовка даних»** → модель Claude: за замовчуванням `claude-sonnet-5`.
  Заміни, якщо потрібна інша.
- Папка збереження вже прописана (ID папки CMR). Змінити — у ноді «Копія шаблону» → options → parents.

## Що варто підтвердити / можливі доопрацювання

- **Пошта:** workflow зроблено під **Gmail** (бо Drive і пошта Unitex — в екосистемі Google).
  Якщо докс-пошта не Gmail — тригер і відповідь треба переробити на **IMAP + SMTP**.
- Мапінг адрес «на всіх у копії» залежить від того, як саме Gmail-нода віддає заголовки —
  можливо, доведеться підправити поля `from`/`cc` у ноді «Підготовка даних» після першого тесту.

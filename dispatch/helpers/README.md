# Допоміжні скрипти / патерни сесії

Ці скрипти НЕ є частиною постійної автоматизації — це разові інструменти, якими користувалися під час сесії. Постійна автоматизація — у `../workflows/`.

## sheets-proxy (тимчасовий n8n-воркфлоу)
Оскільки Google Drive-конектор бачить лише свої файли, а майстер-таблиця у корпоративному акаунті — усі читання/записи в таблицю робилися через тимчасовий n8n-воркфлоу «TEMP Sheets Proxy (Claude)»:
- Webhook `POST /webhook/claude-sheets-proxy` → HTTP Request (raw JSON body, cred googleSheetsOAuth2Api) → Respond.
- Тіло запиту: `{"method":"GET|POST|PUT","url":"<sheets API url>","payload":{...}}`.
- ⚠ HTTP-вузол має слати `contentType: raw` (не «JSON»), інакше n8n шле порожнє тіло.
- Активувати перед серією операцій, деактивувати після (може відпадати за таймаутом).

## Разовий Maersk-перерахунок з кешу
Під час сесії статуси перераховувалися локально з кешованої відповіді Maersk (execution includeData) тим самим мапінгом, що й у `../maersk-tracking/rozbir.code.js`, щоб не робити зайвих API-викликів. Логіка мапінгу — джерело правди в rozbir.code.js.

## Формули Sheets REST у цій таблиці
Розділювач аргументів формул — `;` (локаль), не кома. Conditional format не додавати в тому ж batchUpdate, що й нову колонку (atomic rollback).

## Джерело угод
Експорт з Експедитора Про: журнал «Документи → Угоди» → «Дії → Вивести список» → .mxl (формат 1С, текстовий). Парситься regex-ом. Колонки: 1 Номер, 5 Тип, 6 Статус угоди, 7 Клієнт, 8 POO, 9 POL, 10 POD, 11 FD, 12 Лінія(MAE=Maersk), 13 Коносамент(=BL), 14 Список контейнерів, 15 ЕТА, 16/17/18 К-сть 20/40/інші, 20 Дом.коносамент, 21 Дата завершення(=оплата, НЕ доставка).
